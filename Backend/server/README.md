# Backend (Nest) — Integration & Admin quickstart

This project runs the Nest backend on port 5000 (see `src/main.ts`). The frontend (Next) usually runs on port 3000.

Follow these steps to test the Admin endpoints and connect the frontend correctly:

## 1) Make sure backend is running

```powershell
cd server
npm run start:dev
```

Server will listen on http://localhost:5000

## 2) Apply DB migrations and generate client (if you changed Prisma schema)

```powershell
cd server
npx prisma migrate dev --name add-user-role
npx prisma generate
```

## 3) Create/promote an admin user

- Quick way: use the provided seed script
```powershell
cd server
# optional: ADMIN_EMAIL and ADMIN_PASSWORD env vars
ADMIN_EMAIL=admin@example.com ADMIN_PASSWORD=secret123 npm run seed:admin
```

This will create or update a user with `role = 'ADMIN'` and `isActive = true`.

## 4) Use the token from `/auth/login` (login user) when calling admin endpoints

Admin endpoints (protected by roles):
- POST /admin/products — create product (multipart/form-data)
- GET /admin/products — list products
- GET /admin/products/:id — fetch product
- PUT /admin/products/:id — update
- DELETE /admin/products/:id — delete

Always include header:
Authorization: Bearer <JWT_TOKEN>

## 5) Frontend integration tips

- Ensure the frontend points at the backend base URL. In Next you can set in `.env.local`:

```
NEXT_PUBLIC_API_URL=http://localhost:5000
```

- When running Next.js locally you can add a rewrite so `/api/*` calls in the browser are proxied to the backend (optional)

Add to `next.config.js` in the Next project:

```js
/** @type {import('next').NextConfig} */
module.exports = {
  async rewrites() {
    return [
      { source: '/api/:path*', destination: 'http://localhost:5000/:path*' },
    ];
  },
};
```

Then the front-end can call `/api/admin/products` and the request will go to the backend.

## 6) Postman examples

- Login (get token):
POST http://localhost:5000/auth/login
Body: JSON -> { "email":"admin@example.com","password":"secret123" }

- Then call admin endpoints using that token as `Authorization: Bearer <token>`.

## Description

[Nest](https://github.com/nestjs/nest) framework TypeScript starter repository.

## Project setup

```bash
$ npm install
```

## Compile and run the project

```bash
# development
$ npm run start

# watch mode
$ npm run start:dev

# production mode
$ npm run start:prod
```

## Run tests

```bash
# unit tests
$ npm run test

# e2e tests
$ npm run test:e2e

# test coverage
$ npm run test:cov
```
